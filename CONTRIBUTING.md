# Contributing

Thanks for considering contributing! A few guidelines to keep contributions smooth.

- Open an issue before proposing large changes so we can discuss design.
- Fork and create feature branches. Keep PRs focused and small.
- Follow existing code style; prefer clarity over cleverness.
- Run the dev server locally to test changes: `npm run dev`.
- Include tests or examples for new features when possible.

If you want help getting started, mention which area (editor, transpiler, interpreter, components) and I can suggest a low-risk first issue.
